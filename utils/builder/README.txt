Установить nodejs:
http://stackoverflow.com/a/16303380/1105635
Убедиться, что менеджер пакетов npm так же установился.
# npm -v

Затем установим сборщик Gulp в глобальную область видимости(чтобы просто выполнять в консоли команду gulp где угодно):
# npm install -g gulp

Теперь можно проверить установку gulp:
# gulp -v

Перейдем в дирректорию сборщика (builder).
Теперь нужно установить зависимости для собрщика.
Зависимости лежат в package.json.
Чтобы установить их, достаточно находясь в папке сборщика выполнить:
# npm install

В папку сбощика установятся модули nodejs (node_modules)
Так как нам в проекте эти модули не нужны, чтобы лишний раз не нагружать IDE мы можем попросить исключить эти файалы из проекта.
В IDEA в структуре проекта правй клик по builder/node_modules -> Mark Directory As -> Excluded

Ну и самое главное.
Теперь, мы можем запустить сборщик.
# gulp

Происходит следующее:
путь lessSourceFiles (gulpfile.js) просматривается сборщиком, и компилит CSS;
запускается watcher. Как только меняется любой less-файл по указанному пути или в поддиректории,
то будет собираться файл, подходящий под маску lessSourceFiles;

Возможно, при запуске возникнет ошибка, связанная с модулем node-notifier.
node-notifier - это модуль для вывода уведомлений. Для Win используется Growl(нужно установить),
для Linux notify-send(libnotify-bin, возможно, нужно установить), для Mac ничего не надо.
В собрщике уведомления будут появляться только при неудачной сборке(ошибка в синтаксисе LESS).
Если что-то будет не работать, то можно закомментировать строку 27 (notifier.notify ...) и строку 8 (var notifier ...).

В текущем сборщике настроена сборка less + source map для него.
Если что, то source map позволяют показывать в инспекторе откуда какая директива css взялась,
и указывает она на конкретное месте в less-файле. (Нужно проверить опцию в DevTools "Enable CSS source maps")
